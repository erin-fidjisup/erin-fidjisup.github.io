Web development : Creation of a backlog

Project information

  • Category: Web development
  • Client: Reboot Conseil
  • Project date: Feburary , 2023
  • Technology: VueJS (Vuetify), Vite, NodeJS, AzureDevOps

Creation of a backlog interface to manage category creation and user processing. User and category management using CRUD. Right management since only administrators can have access.

  • Category Management: The interface provided functionality for creating, viewing, updating, and deleting categories. Administrators could easily add new categories, edit existing ones, and remove categories that were no longer needed. This allowed for efficient organization and classification of backlog items based on different categories.
  • User Management: The interface included user management features, allowing administrators to create new user accounts, view user details, update user information, and deactivate or delete user accounts if necessary. User management ensured that the right individuals had access to the backlog interface and maintained data integrity.
  • Access Control and Rights Management: To ensure data security and maintain control over the system, the interface implemented access control mechanisms. Only administrators had access to the backlog interface, and appropriate authentication and authorization measures were put in place to verify their credentials. This ensured that only authorized personnel could perform CRUD operations on categories and manage user accounts.
  • CRUD Operations: The interface provided a user-friendly interface for performing CRUD operations. Administrators could easily create new categories or users, view existing data, update information as needed, and delete categories or deactivate user accounts when required. These operations were performed through intuitive and interactive user interfaces, enhancing the overall user experience.

By creating a backlog interface with category and user management capabilities, we provided administrators with a powerful tool for organizing and maintaining the backlog effectively. The interface ensured secure access control, seamless CRUD operations, and streamlined management of categories and users. Overall, this project demonstrated our proficiency in developing user-friendly interfaces, implementing CRUD functionalities, incorporating access control mechanisms, and utilizing appropriate technologies for efficient category and user management.
