Project information

  • Category: Web development
  • Client: Reboot Conseil
  • Project date: May , 2023
  • Technology: VueJS (Vuetify), Vite, NodeJS, AzureDevOps, Sequelize

Creation of a userfriendly interface to add and remove posts from the favorites. Management of the communication between the service and the backend controller and the display in VUEJS at the front end. Integration also of the notion of extension of model class of Sequelize and use of generics to define attributes of creation of models.
  • User Interface Design: We focused on designing a clean and user-friendly interface for managing favorite posts. This involved creating intuitive user interactions, such as buttons or icons for adding and removing posts from the favorites. We paid attention to visual cues and feedback to provide a seamless experience for users. This part of the project was made with collaborators and design experts
  • Frontend Development with Vue.js: Using Vue.js, we developed the frontend of the application. We implemented the user interface design and integrated it with the backend services. Vue.js allowed us to create reactive components and handle dynamic data updates, providing a smooth and interactive interface for managing favorite posts.
  • Communication with Backend Controller: We established communication between the frontend interface and the backend controller. This involved defining API endpoints and handling requests and responses. We ensured the proper transfer of data between the frontend and backend, enabling seamless interaction and synchronization of favorite posts.
  • Integration of Sequelize Model Class Extensions: In the backend, we leveraged Sequelize, an ORM (Object-Relational Mapping) library, to interact with the database. We extended the Sequelize model classes to include additional functionality and attributes specific to the project requirements. This allowed us to define the structure and behavior of the data models used for storing favorite posts.
  • Use of Generics for Model Attribute Definition: We utilized generics in the model class definitions to define attributes dynamically based on the project's needs. This approach enabled flexibility in defining and managing attributes associated with creating models, allowing for efficient and reusable code.
Throughout the project, we prioritized user experience, ensuring that the interface was user-friendly and intuitive for adding and removing posts from favorites. We paid attention to the communication between the frontend and backend, ensuring seamless data transfer and synchronization. The integration of Sequelize model class extensions and the use of generics enhanced the efficiency and flexibility of the backend implementation.
