Administration system project

Project information

  • Category: Remote Console to help with File Transfer
  • Client: School Project
  • Project date: January , 2023
  • Technology: WPF (Avalonia), C#, .NET, GitHub

The project I undertook involved creating a file transfer application with a remote console using WPF (Windows Presentation Foundation). The primary objective was to develop an application that allowed users to transfer files efficiently while providing a remote console feature for managing file processing. The project utilized the Avalonia framework to ensure cross-platform compatibility, enabling the console to be accessible on different operating systems. The application was built using C# and .NET, leveraging their capabilities to manage file processing, including pause and stop functionalities.

  • File Transfer Functionality: The application provided users with the ability to transfer files between different locations. This involved implementing features such as file selection, source and destination directory specification, and progress tracking during the file transfer process.
  • Remote Console Development: The project incorporated a remote console feature, allowing users to manage file processing remotely. The console provided functionalities such as starting, pausing, and stopping file transfers. The console was designed using WPF, ensuring a visually appealing and intuitive user interface.
  • Avalonia Framework Integration: To ensure cross-platform compatibility, the Avalonia framework was utilized. This allowed the application to run on various operating systems, including Windows, macOS, and Linux. The Avalonia framework provided a unified development experience, enabling the remote console to be available to users regardless of their operating system.
  • Pause and Stop File Processing: The application implemented functionality to pause and stop file transfers. Users could pause a transfer in progress and resume it later, or stop the transfer entirely if needed. This required managing the state of file processing, ensuring proper handling of pause and stop requests.
  • C# and .NET Utilization: The application was developed using C# programming language and the .NET framework. C# provided the necessary tools and libraries to implement file transfer functionality, handle remote console interactions, and manage file processing states. The .NET framework facilitated efficient and reliable file management and allowed for seamless integration of the remote console.
